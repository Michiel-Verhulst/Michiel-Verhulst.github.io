My projects

Observatoire

Project Description

g my internship, I undertook the task of revamping the 'Observatoire' application, which primarily focused on managing document descriptions and their underlying structures. This comprehensive project encompassed redefining the document and structure hierarchies, as well as the associated data migration and website development.

Using ViteJS for the frontend and Spring Boot for the backend, I embarked on a journey spanning approximately 10 weeks. The initial phase involved an in-depth analysis of the existing application and its database structure, laying the foundation for subsequent tasks.

Key features of the application included:

  • Migrating data from the legacy database to a new database system.
  • Designing and developing a user-friendly website interface to interact with the application.
  • Implementing functionalities to display documents, their versions, linked structures, and associated attributes.
  • Enabling users to perform essential actions such as creating, copying, editing, and deleting document versions and structures.
  • Introducing advanced features such as exporting structures to XML format and visualizing structure links.

RitA Coverband

Project Description

As part of my first-year coursework in the PBA Applied Computer Science program, I undertook the RitA Coverband project. This endeavor involved the creation of a PHP-based website for a cover band based in Aarschot, Belgium.

DoctorGPT

Project Description

DoctorGPT is an AI Application designed to provide informative and accurate assistance within seconds, particularly in scenarios where first aid may be needed. The primary objective is to offer step-by-step instructions for first aid, aiming to minimize the effects of health issues and potentially save lives.

The application is intended to be accessible to as many people as possible, with the goal of providing valid information to users worldwide. It is not designed to provide definitive diagnoses but rather to help users identify possible illnesses or diseases based on their symptoms.

DoctorGPT relies on enormous datasets containing symptoms and their potential illnesses or diseases to ensure accuracy in providing information back to the user.

Key features of the application included:

  • Account Creation: Users can create a new account by providing necessary information.
  • Login System: After creating an account, users can log in to access the chat functionality.
  • Symptom Input: Users can input their symptoms into the chat, and the chatbot provides a likely diagnosis.
  • Account Management: Users can manage their accounts, including editing profile details and deleting the account.
  • Edit Account: Users can modify account variables such as length and weight.
  • Delete Account: Users can choose to delete their accounts, with a confirmation step before deletion.

Opvolging Stagebegeleiding

Project Description

The internship within the IT programs at UCLL spans over several weeks. At the beginning of the internship, students receive a task within the field of work, which they then carry out over the course of several weeks. At the end, the process is summarized in an internship report.

The goal of an internship is to provide students with the opportunity to apply what they have learned throughout their education in a realistic work environment, within an existing IT team. To facilitate this integration, it is important for students to regularly seek feedback on their performance.

Two individuals are involved in guiding the student: the internship mentor (from the company) and the internship supervisor (lecturer). Both individuals should be involved in the feedback loop, although it often proves difficult due to their hectic schedules and preference for technical discussions rather than professional skills. However, it is precisely these professional skills that are crucial to further develop.

The process that enables students to receive feedback can be improved. It's up to you to create an application for this purpose

Key features of the application included:

  • Internship coordinators create a guidance matrix (different types of internships).
  • Internship coordinators create an internship (with mentor).
  • Internship coordinators match a student with an internship.
  • Students can initiate a feedback loop starting with a self-evaluation and then evaluated by the internship mentor before finally reaching the internship supervisor.

Minesweeper

Project Description

MineSweeper is a classic game where your goal is to uncover all the squares on the board without detonating any mines. It's a test of logical reasoning and strategy as you navigate through the minefield.

How to Play

Choose Difficulty: Select from Easy, Medium, Hard, or Custom difficulty levels to set the grid size and number of mines.

Game Mechanics:

  • Left-click to uncover a square. Avoid detonating mines!
  • Right-click to flag a potential mine location.

Key features of the application included:

  • Mouse Hover Effects: Visual feedback when hovering over squares.
  • Timer: Track your completion time.
  • Custom Mine Probability: Adjust mine probability for custom difficulty.
  • Quickstart: Jump straight into a new game.
  • Font Color Indicators: Color-coded numbers indicate nearby mines.
  • Exit Game: Start fresh with a new game at any time.

Materialenbank

Project Description

The project I worked on during my second year at UCLL was a web application available on mobile devices. The application aimed to connect various providers, including IMEC, KULeuven, and the City of Leuven, with three different types of users: Vites for furniture, Materialenbank for construction materials, and Scholen 20-30 for educational materials.

Key features of the application included:

  • Each provider and user had their own account.
  • There was one moderator with access to all information.
  • Users could only view information relevant to them.
  • Users could forward certain items to other users.
  • The application included a chat function for users to communicate with providers.
  • Providers were required to include a photo and the date of the last pickup for each item.
  • Interested users received notifications for new items in their categories.
  • There was a validation step for submitted materials and the ability to move items between categories if necessary.
  • The application featured a messaging system between providers and users.
